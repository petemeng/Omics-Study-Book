# 写作约定

这本书面向希望系统学习组学的生命科学读者。写作时优先使用“为什么要做这个组学 -> 核心原理 -> 结果长什么样 -> 这些结果能回答什么生物学问题 -> 高水平论文案例深读 -> 常见误区”的结构。

## 风格

- 中文为主，关键英文术语保留英文。
- 少用口号，多解释为什么。
- 每章至少回答四个问题：为什么要做这门组学、它测到的直接对象是什么、典型结果可以解决什么生物学问题、不能解决什么问题。
- 每个主组学章节开头必须加入“学习导向”块，固定回答：这门组学一般为了解决什么问题；典型分析思路是什么；为什么要按这个顺序分析；背后的核心生物学主线是什么。目标是让读者学会判断，而不是记住工具名。
- 每个主组学章节还应加入“零基础生物学底座”块，用不用专业背景也能理解的语言解释：这门组学测到的生物学对象是什么、它在细胞/组织/生态系统里做什么、为什么它的变化会和表型有关。
- 分析流程必须写成“生物学问题 -> 可观测信号 -> 数据结构 -> 统计/算法动作 -> 可支持的结论 -> 需要验证的边界”。不要只写软件步骤。
- 生物学知识要优先展开：分子/细胞/组织/生态/遗传过程本身是什么，为什么这些过程会在该组学数据里留下可测信号，哪些信号只是间接影子。
- 重要方法要写清楚输入、输出、关键假设和失败场景。
- 尽量加入 Cell / Nature / Science 及其高影响子刊案例。案例不是只列引用，而要拆解：作者为什么必须用该组学、关键结果是什么、结论如何从结果中推出、仍然有哪些局限。
- 涉及植物的组学章节要加入植物提示，例如细胞壁、叶绿体污染、多倍体、根际分层、组织自发荧光等。
- 图形优先使用真实生物学图、结构图、显微图、论文开放许可图或 Wikimedia Commons 图。少用纯文字 Mermaid 图；如果使用 Mermaid，应当用于逻辑链或研究设计，而不是替代分子结构和组织结构图。每张外部图必须标明来源链接、作者/出处和许可；优先选择 CC BY、CC BY-SA、Public Domain、PMC Open Access 或 PLOS 等可复用来源。

## 内容边界

- 这是学习型教材，不替代具体项目的统计咨询、医学诊断或临床决策。
- 工具名和流程可以给出，但重点放在原理和判断逻辑，而不是命令堆砌。
